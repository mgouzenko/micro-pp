Detailed Description

This is the queue used by the webserver and thread pool to enqueue and process requests, respectively.

When the server finishes asynchronously building a complete request, it will enqueue itself on the work_queue. Since the server is a coroutine that consists of nothing more than a few pointers, it can enqueue itself without much overhead. On the other ends, threads from the thread pool dequeue servers from the work_queue and invoke the appropriate callback function on the completed request. In this model, the server is the producer, and the thread pool threads are the consumers.
